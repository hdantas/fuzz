This block type is kind of special in that it is a hybrid between a block and a primitive (it can be fuzzed). The user does not need to be wary of this fact.

__init__(self, block_name, request, offset=0, length=4, endian='<', format='binary', inclusive=False, signed=False, math=None, fuzzable=False, name=None)

source code 

Create a sizer block bound to the block with the specified name. You *can not* create a sizer for any currently open blocks.

Parameters:
  • block_name (String) - Name of block to apply sizer to
  • request (s_request) - Request this block belongs to
  • length (Integer) - (Optional, def=4) Length of sizer
  • offset (Integer) - (Optional, def=0) Offset for calculated size value
  • endian (Character) - (Optional, def=LITTLE_ENDIAN) Endianess of the bit field (LITTLE_ENDIAN: <, BIG_ENDIAN: >)
  • format (String) - (Optional, def=binary) Output format, "binary" or "ascii"
  • inclusive (Boolean) - (Optional, def=False) Should the sizer count its own length?
  • signed (Boolean) - (Optional, def=False) Make size signed vs. unsigned (applicable only with format="ascii")
  • math (Function) - (Optional, def=None) Apply the mathematical operations defined in this function to the size
  • fuzzable (Boolean) - (Optional, def=False) Enable/disable fuzzing of this sizer
  • name (String) - Name of this sizer field
